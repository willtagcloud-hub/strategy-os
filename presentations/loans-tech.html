Slide 04 · PWA

PWA notes

  • Reuses the Visa quiz engine, just a different config.
  • Service worker: never cache /api/match or /api/offers.
  • Strict CSP nonces (no inline scripts) — reduces injected-form fraud.
  • "My Offers" view becomes a returning-user destination; push notifies status updates where buyers permit.
  • Phone verification gate before consent capture; binds the human to the lead.
Slide 05 · Quiz funnel

Soft-qualification flow

  1. Loan purpose (debt, home improvement, auto, medical, life event, other).
  2. Loan amount slider (USD 1k - 100k).
  3. Credit band (excellent / good / fair / poor / not sure).
  4. Annual income.
  5. Employment type and length.
  6. Housing (own/rent) + monthly housing cost.
  7. State / ZIP (or postcode for UK).
  8. Date of birth (gates underage).
  9. Last-4 SSN — masked, encrypted, only sent to buyers requiring it.
  10. TCPA consent block — explicit, recorded, with full creative text.
  11. Phone + email; phone verification SMS code.
  12. (Optional) Plaid bank verification for premium-bid buyers.
Slide 06 · Match engine

Decision flow (target P95 ≤ 1.5 s)

// Cloudflare Worker · POST /api/match
validate(payload)
fraud = botMgmt(payload, headers)
if (fraud > THRESHOLD) return reject

lead = persist(payload)            // includes consent_ledger + R2 audit
candidates = filter(buyers, lead)  // JSON-Logic rules

bids = await Promise.allSettled(
  candidates.map(b =>
    fetchWithTimeout(b.endpoint, mapPayload(b, lead), 800) // ms
  )
)

winner = pickWinner(bids, rules)
persist(bid_results)               // ClickHouse

if (winner) post_lead(winner, lead) // HMAC-signed, TCPA proof attached
sendEvent("lead_submitted", lead)

return offerResponse(winner)
Slide 07 · Buyer config

Buyers as data, not code

-- Postgres
create table buyers (
  id, slug, name, endpoint, auth_type,
  request_template jsonb,
  response_path text,
  rules jsonb,           -- {"min_fico":580,"max_amount":50000,"states":["CA","NY"]}
  active bool, priority int
);

Adding a buyer = inserting a row + mapping its fields. json-logic-js evaluates rules against each lead. No code deploy needed for routing changes.

Slide 10 · Compliance

TCPA + state rules — the most important section

  • Express written consent captured with: user data, exact disclosure text, timestamp, IP, UA.
  • Consent stored in consent_ledger + replicated to R2 write-once bucket.
  • Specific buyer disclosure at consent: every potential buyer named with link.
  • Voice consent path via Twilio <Record>; documented retention.
  • Two-party consent states (CA, FL, ...) → verbal confirmation at call start.
  • One-click opt-out endpoint; daily DNC scrub vs National DNC + internal + partner feeds.
  • NMLS-licensed lenders only (US); FCA partners only (UK); maintain license records.
  • No "guaranteed approval" / "no credit check" copy — counsel-reviewed creatives only.
  • PII column-encrypted with pgsodium; 90-day default retention for non-funded leads.
  • CCPA right-to-opt-out-of-sale wired into opt-out endpoint.
Slide 11 · Callback ops

Default self-serve, callback only on fallback

  • Primary flow: PWA → match → click-out to buyer (no human required).
  • Callback triggers when the lead fails the loan match but qualifies for debt relief or credit monitoring.
  • n8n routes the lead into HubSpot's Debt Relief pipeline.
  • Twilio click-to-call within 60 s of disposition; recordings retained 30 days.
  • State-specific TCPA scripts; never auto-dial without timestamped express consent.
Slide 12 · DevOps

Build, ship, observe

Pipeline

  • GitHub Actions: lint → types → unit → e2e + load (200 RPS sustain) → deploy.
  • Lighthouse PWA ≥ 90 required.
  • Preview env per PR; Supabase branch databases.

SLOs & alerts

  • Match endpoint P95 ≤ 1.5 s; breach for 5 min → page.
  • Per-buyer accept rate < 50% for 1 h → Slack alert.
  • DNC scrub freshness must be < 24 h.
  • Consent ledger weekly checksum; offsite (R2) backup.
  • Daily Postgres logical backup; RPO 15 min, RTO 1 h.
