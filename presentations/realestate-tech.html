Slide 04 · PWA

PWA implementation

  • Manifest: categories: ["productivity","finance","lifestyle"]. Installable; install retains buyers across the multi-month decision window.
  • Service worker: pre-cache app shell + fonts; never cache /api/match/*, /api/affordability, or /api/click.
  • Background Sync for partial-quiz submissions + saved searches.
  • Push topics: saved_search_alert, rate_change, agent_followup, bundle_offer. Quiet hours per timezone.
  • Strict CSP nonces; encrypted IndexedDB for partial quizzes + saved searches.
Slide 05 · Buyer-readiness quiz

Quiz flow (≤ 90s)

  1. Stage (just looking · 12+ mo · 3-12 mo · ready now)
  2. City / metro (Smarty autocomplete)
  3. Bedrooms + property type
  4. Approximate household income
  5. Approximate down payment
  6. First-time buyer (gates first-time programs)
  7. Pre-approval status (none · soft · full)
  8. Timeline + commute (optional)
  9. Email · name · phone
  10. Fair Housing-compliant consent (no protected-class questions)

Output: affordability snapshot (uses daily rate feed) → 3-5 agent shortlist + 3-5 mortgage shortlist → buyer-journey tracker with saved-search + price-band alerts.

Slide 07 · Capacity waterfall

Match logic + waterfall

// POST /api/match/agent
candidates = agents.filter(a => a.metros.has(metro) && a.active)
candidates = candidates.filter(a =>
  assignments(a, today) < a.capacity_per_day)
ranked = candidates.sort(by rating desc, random tie-break)
return top 3-5

// If empty: fall back concentric ZIPs
if (ranked.length === 0) {
  candidates = agents in adjacent ZIPs
}

// If still empty: queue + ops alert
if (ranked.length === 0) {
  mark lead "queued"
  Slack alert ops; do NOT silently drop
}

// Mortgage waterfall mirrors this, gated by NMLS state
  • Goal: keep per-partner accept rate ≥ 80%.
  • Capacity caps prevent overwhelming partners; rolling daily counts in agent_capacity_log.
  • NMLS validation is a hard precondition for mortgage matches.

Slide 09 · Tracking

Server-side conversions (SAC-compliant)

view_city_page → quiz_started → quiz_completed (metro, stage)
→ match_generated_agent → match_generated_mortgage
→ click_to_agent → click_to_mortgage
→ agent_accepted          // primary CPL (agent)
→ mortgage_pre_qualified  // primary CPL (mortgage)
→ mortgage_funded         // high-value CPA
→ deal_closed             // rev-share trigger
→ saved_search_alert_open → optout
  • Meta SAC: hashed identifiers only; no protected attributes; SAC-compliant audiences and broad targeting.
  • Optimize on quiz_completed initially → switch to agent_accepted / mortgage_pre_qualified at sufficient volume.
  • Google Ads: Enhanced Conversions for Leads with mortgage_funded value-based bidding.
  • Daily reconciliation vs partner reports; > 5% variance → Slack alert.
Slide 10 · Buyer journey

The retention layer

  • Saved search alerts → new listings or price drops trigger Klaviyo email + push.
  • Rate alerts → 30-yr fixed moves > 0.25% from saved snapshot → push notification.
  • Drip series → day 0 / 3 / 7 / 14 / 30 / 60 / 90 educational content (closing-cost myths, inspection checklist, neighborhood guides).
  • Concierge ping → if user marked "ready now" but hasn't clicked an agent by day 30, HubSpot task assigned to concierge.
  • Cancel-intent flow → opt-out path offers "pause for 30 days" instead of full unsubscribe.
  • Programmatic city SEO compounds organic returns alongside paid (Phase 2+).
Slide 11 · Compliance

SAC + Fair Housing + NMLS guardrails

  • Meta Special Ad Category (US Housing): compliant audiences, no age/gender/ZIP-radius targeting, no certain lookalikes.
  • Fair Housing Act: no discriminatory copy or targeting; counsel-reviewed creative bank.
  • State licensing of agents: verify at onboarding + monthly recheck (state DOI APIs where available).
  • NMLS: verify mortgage partner NMLS at onboarding + monthly recheck.
  • TCPA: express consent for SMS/calls; daily DNC scrub vs National + internal + partner feeds.
  • CCPA / GDPR: DSAR + opt-out endpoints; right-to-erase ≤ 30 days.
  • PII columns encrypted with pgsodium; 90-day default retention for unconverted leads.
  • Two-party consent disclosure on concierge calls (CA, FL, etc.).
  • Rate snapshots dated; APR transparency on every CTA.
  • Per-vertical brand domain; warmed accounts; pixel separation.
Slide 12 · DevOps & SLOs

Operational posture

SLOs

  • /api/match/agent P95 ≤ 800 ms
  • Daily rate ingest by 9:00 ET; alert on staleness > 3 h
  • Per-partner accept rate < 50% for 1 h → Slack page
  • Saved search alert delivery ≥ 99%
  • Reconciliation variance > 5% → Slack page

Observability + flags

  • Sentry FE/BE + Vercel Analytics RUM
  • Better Stack uptime on /api/match/*, /api/click, /api/webhooks/*
  • Logflare → BigQuery warehouse
  • Flagsmith for capacity rules, drip variants, city templates
  • GitHub Actions: lint → types → unit → e2e → Lighthouse PWA ≥ 90

Runbooks: agent capacity exhaustion, rate feed lag, postback gap, SAC audit.
