Slide 04 · PWA

Per-line PWA paths

  • One framework, per-line landing pages: /quote/auto, /quote/home, /quote/life, /quote/health.
  • Medicare lives on a separate subdomain with CMS-compliant copy and SOA capture before plan-specific content.
  • Same quiz engine as Visa/Loans/GLP-1 with per-line configs.
  • Strict CSP nonces; never cache /api/match, /api/call, or quote results.
  • Click-to-call CTA prominence on result pages: tel: link + server-pushed warm-transfer flow when an agent is online.
Slide 05 · Match flows

Form match + warm-transfer call

// POST /api/match (form lead)
validate → fraud_score (Twilio Lookup + Bot Mgmt)
lead = persist(payload + consent)
candidates = filter(buyers, lead)
bids = Promise.allSettled(parallel pings, 800ms timeout)
winner = pickWinner(bids, rules)
if winner: post_lead(winner)
sendEvent("lead_submitted")

// POST /api/call/request (warm transfer)
agent_pool = pickPool(line, state, hours)
twilio.studio.execute("warm_transfer_flow", { lead, agent_pool })
//   1. Dial agent → press 1 to confirm
//   2. Bridge consumer + agent
//   3. Record (with disclosure)
//   4. Disposition recorded via webhook
sendEvent("call_initiated")
Slide 06 · Buyers

Buyers as data, not code

-- Postgres
create table buyers (
  id, slug, name, line, endpoint, auth_type,
  request_template jsonb,
  response_path text,
  rules jsonb,
  payout_model text,         -- ping_post | call | hybrid
  per_call_payout_cents int,
  active bool, priority int
);
  • JSON-Logic evaluates rules against each lead (line, state, age, etc.).
  • Pay-Per-Call: per-buyer Twilio DIDs; payout triggers when call duration ≥ buyer's billable threshold.
  • Adding a buyer or line = config insert + map fields. No code deploy.

Slide 10 · AEP playbook

Annual Enrollment Period (Oct 15 - Dec 7)

  • Capacity: 3-5x normal traffic; surge agent pool via partner call center.
  • Creative bank: CMS-compliant copy bank prepared by Sept 1; counsel-reviewed.
  • SOA capture: mandatory before plan-specific discussion; stored in consent_ledger with type=soa.
  • Routing: in-house agents first, partner overflow second; tracked separately.
  • Real-time dashboard for daily ops review during AEP.
  • Post-AEP plan: shift budget to Auto/Home/Life starting Dec 8 to smooth seasonality.
Slide 11 · Compliance

TCPA + state + CMS guardrails

  • Express written consent captured with: user data, exact disclosure text, timestamp, IP, UA.
  • Consent stored in consent_ledger + replicated to R2 write-once.
  • Specific buyer disclosure at consent: every potential buyer named with link.
  • Voice consent path via Twilio <Record>; documented retention.
  • Two-party consent states (CA, FL, ...) → verbal disclosure at call start.
  • One-click opt-out + daily DNC scrub vs National DNC + internal + partner feeds.
  • State licensing verification of partners pre-launch + quarterly recheck.
  • Medicare creative review + SOA + no misleading TPMO claims; separate brand domain.
  • Per-line ad accounts; pixel separation; warmed accounts.
  • Pen test mandatory pre-launch and annually.

Top risks

  • TCPA litigation → counsel-reviewed disclosures, immutable ledger, voice consent.
  • CMS / Medicare scrutiny → separate brand, SOA capture, every Medicare creative reviewed.
  • Buyer policy changes → 4-6 buyers per line, dynamic routing.
  • Lead fraud → Cloudflare Bot Mgmt + Twilio Lookup + phone verify.
  • Post-AEP traffic crash → mix lines (Auto / Home / Life) for seasonality smoothing.
