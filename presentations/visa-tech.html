Slide 04 · PWA

PWA implementation specifics

Manifest + install

  • Standalone, portrait, theme-locked.
  • Custom install prompt at quiz step 4.
  • iOS Safari fallback banner with manual instructions.
  • Track appinstalled as a server-side event.

Service worker (Workbox)

  • Pre-cache app shell + fonts.
  • Network-first for /api/*, 3 s timeout.
  • Background Sync for /api/lead on flaky 4G.
  • IndexedDB resume of quiz drafts.
  • Push topics: case-reminders, doc-missing, appt-soon.
Slide 05 · Funnel engine

Quiz engine (config-driven, reused across verticals)

type QuizStep =
  | { id; type:"single";  question; options:[{value,label,weight?}] }
  | { id; type:"multi";   question; options:[{value,label,weight?}] }
  | { id; type:"number";  question; min; max; weight?:(n)=>number }
  | { id; type:"date";    question }
  | { id; type:"branch";  condition; ifTrue; ifFalse }

type QuizConfig = { id; locale; steps:QuizStep[]; scoring:Rule[]; cta }

Scoring runs client-side for instant UX and re-runs server-side on /api/score for trust + persistence. Same engine drives Loans, Insurance, Real Estate.

Slide 08 · Tracking

Server-side conversions

One normalized event taxonomy fans out to every platform with hashed user data and a shared event_id for browser-pixel dedup.

view_prelander → quiz_start → quiz_completed
→ lead_submitted → booking_scheduled → booking_attended
→ package_purchased → case_started → case_milestone_reached
→ partner_handoff_sent → partner_handoff_qualified → partner_handoff_paid
  • Meta CAPI: optimize on lead_submitted → switch to package_purchased at ≥50/wk.
  • Google Ads Enhanced Conversions for Leads: gclid + hashed PII; value-based bidding.
  • TikTok Events API: same payload schema.
  • Retries on QStash; dead letter in server_events.

Slide 10 · Security & compliance

Controls

  • TLS 1.3 only, HSTS preload, strict CSP nonces.
  • Default-deny RLS; service-role only for ingestion.
  • pgsodium column encryption for PII (passport, phone, etc.).
  • Signed-URL document storage (R2), 5-min default expiry.
  • Consent log captured at every PII collection point.
  • GDPR + India DPDP DSAR endpoints; right-to-erase ≤ 30 d.
  • Per-vertical brand domains; no cross-pollination of pixels.
  • Quarterly secret rotation; Dependabot + weekly pnpm audit.
  • Pen test before launch + annually thereafter.
Slide 11 · DevOps

Build, ship, observe

Pipeline

  • GitHub Actions: lint → types → unit → e2e (Playwright on staging) → deploy.
  • Required checks include Lighthouse PWA ≥ 90.
  • Preview env per PR (Vercel + Supabase branch).
  • Feature flags via Flagsmith self-hosted.

Observability

  • Sentry (FE + BE), Vercel Analytics RUM.
  • Better Stack uptime on 10 endpoints.
  • Logflare → BigQuery warehouse.
  • Slack-paged alerts: 5xx spike, conv. drop > 30% vs 7d avg, payment webhook failure.
  • Runbooks in /ops/runbooks/*.
